# 一 软件设计方法论（原则）
1. 概述：
- 模块化
- 接口
- 信息隐藏
- 增量开发
- 抽象
- 一般化

2. 模块化:
将一个系统拆分成多个模块，要求低耦合、高内聚，减少出错提高可维护性。

3. 接口：
信息隐藏通过接口的定义来完成。信息隐藏越多，耦合性越低。

4. 增量开发：
以模块化为基础；明确的接口定义为增量开发提供可能。

5. 抽象：
同层级放在函数调用前，子层级放在函数调用内部。
